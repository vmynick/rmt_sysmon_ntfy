Install one command

Telepítés egy parancs

A single line. It informs you first, then asks for the topic and language if needed (Enter = default) — everything else it handles itself: python3, copy, systemd service, start.

Egyetlen sor. Tájékoztat, majd ha kell, megkérdezi a topicot és a nyelvet (Enter = alapérték) — minden mást maga intéz: python3, másolás, systemd service, indítás.

curl -fsSL https://raw.githubusercontent.com/vmynick/rmt_sysmon_ntfy/main/install.sh | bash

Fully unattended (e.g. many machines, from a script):

Teljesen kérdés nélkül (pl. több gépre, scriptből):

SYSMON_TOPIC=sysmon-aa0ca9c635659f04 SYSMON_LANG=hu \ curl -fsSL https://raw.githubusercontent.com/vmynick/rmt_sysmon_ntfy/main/install.sh | bash

Extra tasks your own checks

Extra taskok saját ellenőrzések

Write your own checks in the extra_tasks() function. Built-in helpers check_service() and check_docker() are included; they are appended to the end of the status report.

Az extra_tasks() függvénybe írhatsz saját ellenőrzéseket. Beépített segédek: check_service() és check_docker(); a status riport végéhez fűződnek.

# sysmon.py def extra_tasks(): results = [] results.append(check_service("nginx")) # systemd unit up? results.append(check_docker("homeassistant")) # container running? return results

Priority by severity

Prioritás súlyosság szerint

The disk, mem and temp thresholds (and the severity of the extra tasks) determine the ntfy push priority — the phone is silent, loud or alarming accordingly.

A disk, mem és temp küszöbök (és az extra taskok severity-je) határozzák meg az ntfy push prioritását — a telefon ennek megfelelően néma, hangos vagy riasztó.

LevelSzint ThresholdKüszöb ntfy priorityntfy prio PhoneTelefon
okdisk<80% · mem<85% · temp<70°defaultquiet notificationhalk értesítés
warndisk≥80% · mem≥85% · temp≥70°high ⚠️loudhangos
critdisk≥92% · mem≥95% · temp≥80°urgent 🚨alarming, breaks through silent moderiasztó, áttör a néma módon

The thresholds can be set in the TH dict in sysmon.py.

A küszöbök a sysmon.py TH szótárában állíthatók.

Proactive alerts watchdog

Proaktív riasztás watchdog

The daemon doesn't only answer commands. Every SYSMON_INTERVAL seconds (default 300, 0 disables) it re-checks status and pushes only when the severity level changes — a high/urgent alert on degrade, a quiet "recovered" back to ok. No spam while everything is fine.

A daemon nem csak válaszol. Minden SYSMON_INTERVAL másodpercben (alap 300, 0 = ki) újraellenőrzi a státuszt, és csak akkor küld push-t, ha a súlyossági szint változik — high/urgent riasztás romláskor, halk „helyreállt" ha visszaáll. Nincs spam, amíg minden rendben.

Loop prevention reads and writes the same topic

Loop-prevenció ír és olvas ugyanarra a topicra

1self-tag — every push it sends carries a sysmon-<host> tag; incoming events with that tag are skipped → it never answers itself.— minden saját push sysmon-<host> címkét kap; a beérkező eseményeket e címke alapján kihagyja → nem válaszol önmagának.
2allowlist — only the 8 known commands run, everything else is dropped.— csak a 8 ismert parancs fut le, minden más eldobva.
3dedup — the same command at most once per 4 s.— ugyanaz a parancs 4 mp-en belül egyszer.
4rate limit — max 8 outgoing replies / 60 s.— max 8 kimenő válasz / 60 mp.
