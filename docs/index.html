What is ntfy.sh the transport

Mi az az ntfy.sh a csatorna

ntfy.sh is a free, open-source pub/sub notification service. You publish a message to a topic with a plain HTTP POST (e.g. curl), and anyone subscribed to that topic — phone app, browser, or another script — receives it instantly. No account, no API key: the topic name is the only thing you need (and your only secret). sysmon uses it both ways: it pushes status to a topic and subscribes to the same topic to receive commands. You can use the public ntfy.sh server or self-host your own.

Az ntfy.sh egy ingyenes, nyílt forráskódú pub/sub értesítő szolgáltatás. Egy üzenetet egy topicra küldesz sima HTTP POST-tal (pl. curl), és bárki, aki feliratkozott arra a topicra — telefonos app, böngésző vagy másik szkript — azonnal megkapja. Nincs fiók, nincs API-kulcs: csak a topic neve kell (és ez az egyetlen titkod). A sysmon mindkét irányba használja: push-ol egy topicra, és feliratkozik ugyanarra, hogy parancsokat fogadjon. Használhatod a nyilvános ntfy.sh szervert, vagy üzemeltethetsz sajátot.

Install one command

Telepítés egy parancs

A single line. It informs you first, then asks for the topic and language if needed (Enter = default) — everything else it handles itself: python3, copy, systemd service, start.

Egyetlen sor. Tájékoztat, majd ha kell, megkérdezi a topicot és a nyelvet (Enter = alapérték) — minden mást maga intéz: python3, másolás, systemd service, indítás.

Fully unattended (e.g. many machines, from a script):

Action buttons tap, don't type

Akciógombok koppints, ne gépelj

ntfy notifications can carry up to 3 buttons, set with the Actions header. Three types: view (open a URL), http (fire a background request), broadcast (Android intent). sysmon attaches http buttons that POST a command back to your topic — so tapping Top just sends top and you get a fresh reply, no typing. clear=true dismisses the notification after the tap.

Az ntfy értesítések max 3 gombot hordozhatnak, az Actions headerrel beállítva. Három típus: view (URL megnyitása), http (háttér-kérés), broadcast (Android intent). A sysmon http gombokat tesz fel, amik egy parancsot POST-olnak vissza a topicodra — a Top gomb elküldi a top-ot, és kapsz friss választ, gépelés nélkül. A clear=true a koppintás után eltünteti az értesítést.

# view button — open a page# view gomb — oldal megnyitása curl -H "Actions: view, Open docs, https://vmynick.github.io/rmt_sysmon_ntfy/" \ -d "deployed" ntfy.sh/mytopic # http button — send a command back (what sysmon does)# http gomb — parancs visszaküldése (ezt csinálja a sysmon) curl -H "Actions: http, Top, https://ntfy.sh/mytopic, method=POST, body=top, clear=true" \ -d "pi4 status" ntfy.sh/mytopic

Max 3 buttons · header must be one line · quote labels/values that contain a comma. The JSON publish form (actions: [ … ]) avoids the escaping. See the ntfy action-buttons docs.

Max 3 gomb · a header egy sor legyen · idézőjelezd a vesszőt tartalmazó címkét/értéket. A JSON-os küldés (actions: [ … ]) megspórolja az escapelést. Lásd az ntfy akciógomb-dokumentációt.

Extra tasks your own checks

Extra taskok saját ellenőrzések

# ⭐ the easy way — interactive picker, anytime# ⭐ az egyszerű út — interaktív választó, bármikor sudo sysmon configure

sudo sysmon configure lists every running docker container and systemd service — plus Proxmox VMs/CTs on a PVE node (e.g. your Home Assistant OS VM) — marks your current picks, and lets you re-select (number/name, * = all, - = none). It then saves and restarts the service — no file editing. The same wizard runs at the end of the installer. Picks live in SYSMON_CHECK_SERVICES / SYSMON_CHECK_DOCKER / SYSMON_CHECK_PVE and run on every status. For custom logic you can still edit extra_tasks() directly (helpers check_service() / check_docker() / check_pve_guest()).

A sudo sysmon configure listázza az összes futó docker konténert és systemd service-t — Proxmox node-on a VM-eket/CT-ket is (pl. a Home Assistant OS VM-et) —, megjelöli a jelenlegi választást, és újraválaszthatsz (szám/név, * = mind, - = egyik se). Aztán elmenti és újraindítja a service-t — fájl szerkesztése nélkül. Ugyanez a varázsló fut a telepítő végén is. A választás a SYSMON_CHECK_SERVICES / SYSMON_CHECK_DOCKER / SYSMON_CHECK_PVE változókban van, és minden status-nál fut. Egyedi logikához az extra_tasks() közvetlenül is szerkeszthető (segédek: check_service() / check_docker() / check_pve_guest()).

# sysmon.py def extra_tasks(): results = [] results.append(check_service("nginx")) # systemd unit up? results.append(check_docker("homeassistant")) # container running? return results

Priority by severity

Prioritás súlyosság szerint

The disk, mem and temp thresholds (and the severity of the extra tasks) determine the ntfy push priority — the phone is silent, loud or alarming accordingly.

A disk, mem és temp küszöbök (és az extra taskok severity-je) határozzák meg az ntfy push prioritását — a telefon ennek megfelelően néma, hangos vagy riasztó.

LevelSzint ThresholdKüszöb ntfy priorityntfy prio PhoneTelefon

The thresholds can be set in the TH dict in sysmon.py.

A küszöbök a sysmon.py TH szótárában állíthatók.

Proactive alerts watchdog

Proaktív riasztás watchdog

The daemon doesn't only answer commands. Every SYSMON_INTERVAL seconds (default 300, 0 disables) it re-checks status and pushes only when the severity level changes — a high/urgent alert on degrade, a quiet "recovered" back to ok. No spam while everything is fine.

A daemon nem csak válaszol. Minden SYSMON_INTERVAL másodpercben (alap 300, 0 = ki) újraellenőrzi a státuszt, és csak akkor küld push-t, ha a súlyossági szint változik — high/urgent riasztás romláskor, halk „helyreállt" ha visszaáll. Nincs spam, amíg minden rendben.

Two dampeners: flap protection (a non-crit change must persist two checks; crit alerts at once) and quiet hours — set SYSMON_QUIET=22:00-07:00 to hold warn alerts overnight (they fire when the window ends; crit always breaks through).

It also watches GitHub for new releases — on start, then every SYSMON_UPDATE_CHECK seconds (default daily, 0 = off). An "update available" notice is tappable (opens the docs) and carries Dismiss / Remind 6h / Remind 2d buttons. Ask any time with checkupdate. Upgrading = re-run the installer; it never self-updates from a command.

Két csillapító: flap-védelem (egy nem-kritikus változásnak két ellenőrzésig ki kell tartania; a crit azonnal jelez) és csendes órák — a SYSMON_QUIET=22:00-07:00 éjszakára visszatartja a warn riasztásokat (a window végén jeleznek; a crit mindig áttör).

A GitHubot is figyeli új kiadás után — indításkor, majd SYSMON_UPDATE_CHECK másodpercenként (alap napi, 0 = ki). A „frissítés elérhető" értesítés koppintható (megnyitja a dokumentációt), és Dismiss / Remind 6h / Remind 2d gombokat hord. Bármikor kérdezhetsz a checkupdate-tel. Frissítés = futtasd újra a telepítőt; parancsból sosem frissíti magát.

Loop prevention reads and writes the same topic

Loop-prevenció ír és olvas ugyanarra a topicra

1self-tag — every push it sends carries a sysmon-<host> tag; incoming events with that tag are skipped → it never answers itself.— minden saját push sysmon-<host> címkét kap; a beérkező eseményeket e címke alapján kihagyja → nem válaszol önmagának.
2allowlist — only the known commands run, everything else is dropped.— csak az ismert parancsok futnak, minden más eldobva.
3dedup — the same command at most once per 4 s.— ugyanaz a parancs 4 mp-en belül egyszer.
4rate limit — max 8 outgoing replies / 60 s.— max 8 kimenő válasz / 60 mp.
